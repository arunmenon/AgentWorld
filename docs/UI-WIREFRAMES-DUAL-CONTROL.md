## 6. Component Specifications

> **Note:** These are **proposed** component interfaces for new dual-control features.
> Consult the actual `web/src/components` directory for existing patterns before
> implementing. These specs define the expected behavior, not the final file paths.

### 6.1 AccessTypeSelector Component

```tsx
interface AccessTypeSelectorProps {
  value: 'shared' | 'role_restricted' | 'per_agent';
  onChange: (value: AccessType) => void;
  disabled?: boolean;
}

// Visual states:
// - Default: All options visible, none selected
// - Selected: Highlighted with filled radio button
// - Disabled: Grayed out, not interactive
// - With tooltip: Help icon shows explanation on hover
```

### 6.2 RoleSelector Component

```tsx
interface RoleSelectorProps {
  value: 'peer' | 'service_agent' | 'customer';
  onChange: (role: AgentRole) => void;
  showAccessibleApps?: boolean;
  apps?: AppDefinition[]; // For filtering
}

// Behavior:
// - Dropdown with role options
// - Shows icon + name for each role
// - Optionally shows accessible apps below
// - Apps auto-filter when role changes
```
