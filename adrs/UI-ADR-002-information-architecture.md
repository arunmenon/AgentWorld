# UI-ADR-002: Information Architecture & Navigation

## Status
Accepted

## Dependencies
- **ADR-007**: Visualization Strategy (dual CLI/Web requirement)
- **ADR-008**: Persistence & State Management (URL-based state, shared SQLite)
- **ADR-009**: Use Case Scenarios (user workflows for product testing, data generation)
- **UI-ADR-001**: Design System (visual foundation)

## Context

AgentWorld serves four distinct user personas with different jobs-to-be-done:

### User Personas (from UI Spec)

| Persona | Role | Primary Jobs | Key Screens |
|---------|------|--------------|-------------|
| **Dr. Maya** | AI Researcher | Design experiments, analyze emergence, publish findings | Experiment Builder, Analysis View |
| **Alex** | Product Manager | Gather user feedback, test features, make decisions | Focus Group View, Results Export |
| **Sam** | Data Engineer | Generate training data, validate quality, export datasets | Data Generation, Export Tools |
| **Jordan** | Developer | Integrate with systems, debug agents, extend capabilities | Inspector, Config Editor |

### Navigation Requirements

1. **Simulation-centric**: Most workflows revolve around a specific simulation
2. **Deep linking**: Share specific states (simulation at step 45, agent X's memory)
3. **Parallel views**: Compare simulations, view multiple agents
4. **Progressive disclosure**: Simple start â†’ advanced features
5. **State preservation**: Return to exact state after navigation

### Existing Patterns Analysis

| Pattern | Used By | Pros | Cons |
|---------|---------|------|------|
| Single-page dashboard | Grafana | All data visible | Overwhelming |
| Hierarchical navigation | GitHub | Clear structure | Deep nesting |
| Command palette | VSCode, Linear | Fast access | Learning curve |
| Tab-based workspaces | Browser, IDE | Parallel work | Tab overload |

## Decision

Implement a **hybrid navigation architecture** combining hierarchical structure with command palette for power users, and deep-linkable URL schema for state sharing.

## Consequences

### Positive

- **Deep linking** enables sharing exact simulation states for collaboration
- **Command palette** accelerates expert workflows (2-3 keystrokes vs multiple clicks)
- **Progressive disclosure** keeps interface simple for new users
- **URL-based state** survives page refresh, enables browser history
- **Three-column layout** matches researcher mental model (agents â†’ activity â†’ details)
- **CLI parity** ensures consistent access regardless of interface

### Negative

- **Complex URL schema** may be overwhelming in address bar
- **Three-column layout** requires larger screens for full experience
- **Command palette learning curve** for non-power users
- **State sync complexity** between URL, React state, and SQLite

### Risks

- URL schema changes break bookmarked deep links (mitigate with redirects)
- Over-reliance on keyboard shortcuts excludes some users (mitigate with discoverability)
