# UI-ADR-006: Persona Builder & Library

## Status
Accepted

## Dependencies
- **ADR-004**: Trait Vector Persona System (data model for personas)
- **ADR-003**: Multi-Provider LLM Architecture (prompt preview generation)
- **UI-ADR-001**: Design System (TraitSlider component, visual styling)
- **UI-ADR-002**: Information Architecture (navigation to /personas routes)

## Context

Persona creation is a **primary workflow** for three of four user personas:

| User Persona | Persona Builder Usage |
|--------------|----------------------|
| **Dr. Maya** (Researcher) | Creates diverse agent populations for experiments, needs statistical distribution controls |
| **Alex** (PM) | Builds focus group personas matching target demographics |
| **Sam** (Data Engineer) | Generates varied personas for synthetic data quality |
| **Jordan** (Developer) | Less frequent, uses for debugging specific agent behaviors |

### Requirements from UI Spec

1. **3-Step Wizard Flow**: Traits â†’ Identity â†’ Preview
2. **Population Generator**: Batch creation with diversity constraints
3. **Live Prompt Preview**: Real-time view of how traits translate to LLM prompts
4. **Library Management**: Search, filter, duplicate, version personas
5. **Import/Export**: JSON and CSV format support

### Current Gap

UI-ADR-001 specifies the TraitSlider component but lacks the complete persona creation flow, library management interface, and population generation tools.

## Decision

Implement a comprehensive **Persona Builder & Library system** with wizard-based creation, batch generation, and full library management capabilities.

## Consequences

### Positive

- **Streamlined persona creation** via 3-step wizard reduces cognitive load
- **Population generator** enables rapid creation of diverse agent groups
- **Live prompt preview** provides transparency into LLM behavior
- **Library management** supports reuse and organization
- **Import/Export** enables sharing and backup
- **AI assistance** lowers barrier for non-expert users

### Negative

- **Wizard pattern** may feel slower for expert users (mitigate with keyboard shortcuts)
- **Population generator** complexity may overwhelm new users (mitigate with presets)
- **AI-generated personas** require LLM calls, adding cost

### Implementation Notes

1. **Trait Sliders**: Use existing TraitSlider from UI-ADR-001
2. **Prompt Preview**: Server-side generation via ADR-003 LLM provider
3. **Population Generator**: Use numpy/scipy for statistical distributions
4. **Storage**: Personas stored in SQLite per ADR-008
