# UI-ADR-003: Simulation Control Interface

## Status
Accepted

## Dependencies
- **ADR-011**: Simulation Runtime & Scheduling (step semantics, pause/resume, determinism)
- **ADR-008**: Persistence & State Management (checkpoints for pause/resume)
- **ADR-003**: Multi-Provider LLM Architecture (cost tracking, provider selection)
- **UI-ADR-001**: Design System (visual styling for controls)
- **UI-ADR-002**: Information Architecture (control placement in layout)

## Context

AgentWorld simulations require precise control mechanisms that allow researchers to:

1. **Start/Stop/Pause** simulations at any point
2. **Step through** interactions one at a time for analysis
3. **Scrub timeline** to review past states
4. **Inject stimuli** to observe agent reactions
5. **Monitor costs** in real-time to manage budgets
6. **Export results** at any point

### Control Requirements (from ADR-011)

| Requirement | Implementation Need |
|-------------|---------------------|
| Step-by-step execution | Step button, auto-advance toggle |
| Pause/resume | Persistent state via checkpoints |
| Determinism | Seed display and reset capability |
| Cost limits | Budget gauge with warnings |
| Cancellation | Immediate stop with state preservation |

### User Scenarios

**Dr. Maya (Researcher):** "I need to pause at step 47 where the interesting behavior emerged, inject a provocative question, and observe the divergence."

**Alex (PM):** "I want to run the focus group, but stop automatically if we hit the $5 budget limit."

**Jordan (Developer):** "I need to step through slowly to debug why agent Lisa isn't responding to messages."

## Decision

Implement a **comprehensive control system** with timeline-based navigation, real-time cost monitoring, and stimulus injection capabilities.

## Consequences

### Positive

- **Precise control** over simulation execution for research workflows
- **Budget management** prevents unexpected costs
- **Checkpoint system** enables experimentation without risk
- **Stimulus injection** supports interactive research
- **Keyboard shortcuts** enable power-user workflows
- **Visual feedback** keeps users informed of simulation state

### Negative

- **Complex state machine** requires careful implementation
- **Checkpoint storage** adds disk space requirements
- **Many control options** may overwhelm new users (mitigate with progressive disclosure)

### Implementation Notes

1. **State Management**: Use React Context or Zustand for control state
2. **WebSocket Integration**: Control commands send via WebSocket (ADR-012)
3. **Optimistic Updates**: Show UI state change immediately, reconcile with server
4. **Debouncing**: Rapid keyboard input should debounce API calls
