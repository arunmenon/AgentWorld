# UI-ADR-007: Results Analysis & Export

## Status
Accepted

## Dependencies
- **ADR-010**: Evaluation & Metrics System (extractors, validators, metrics)
- **ADR-009**: Use Case Scenarios (product testing, data generation outputs)
- **ADR-008**: Persistence & State Management (stored results)
- **UI-ADR-001**: Design System (charts, colors, components)
- **UI-ADR-003**: Simulation Control (transition to results view)

## Context

Results analysis is the **terminal step of every simulation** and the primary value delivery point:

| User Persona | Results Analysis Needs |
|--------------|----------------------|
| **Dr. Maya** (Researcher) | Statistical metrics, theme extraction, exportable data for papers |
| **Alex** (PM) | Key findings summary, sentiment analysis, shareable reports |
| **Sam** (Data Engineer) | Bulk export to HuggingFace format, quality metrics |
| **Jordan** (Developer) | Raw data access, debugging insights |

### Requirements from UI Spec

1. **5-Tab Results Dashboard**: Overview, Opinions, Themes, Transcripts, Export
2. **AI-Extracted Key Findings**: Automatic insight generation
3. **Sentiment Analysis**: Per-question and per-agent visualizations
4. **Theme Clustering**: Semantic similarity groupings
5. **Multiple Export Formats**: JSON, CSV, HuggingFace datasets, publication-ready

### Current Gap

UI-ADR-003 covers the transition to results view but lacks the complete results dashboard, analysis tools, and export specifications.

## Decision

Implement a comprehensive **Results Analysis Dashboard** with AI-powered insights, interactive visualizations, and flexible export options.

### 9. Component Specifications

```typescript
// Results dashboard state
interface ResultsState {
  simulationId: string;
  activeTab: 'overview' | 'opinions' | 'themes' | 'transcripts' | 'export';

  // Computed analysis (from ADR-010)
  keyFindings: KeyFinding[];
  opinions: AgentOpinion[];
  themes: ThemeCluster[];
  sentimentByQuestion: Record<string, number>;

  // Export state
  exportConfig: ExportConfig;
  isExporting: boolean;

  // Filter state
  selectedAgents: string[];
  selectedQuestions: string[];
  searchQuery: string;
}

interface KeyFinding {
  id: string;
  title: string;
  description: string;
  confidence: number;       // 0-1
  supportingQuotes: Quote[];
  relatedThemes: string[];
}

interface AgentOpinion {
  agentId: string;
  questionId: string;
  stance: 'positive' | 'negative' | 'neutral' | 'mixed';
  confidence: number;
  summary: string;
  keyPoints: string[];
  quotes: Quote[];
}

interface ThemeCluster {
  id: string;
  name: string;
  description: string;
  messageCount: number;
  agents: string[];
  sentiment: number;
  keyTerms: string[];
  representativeQuotes: Quote[];
  relatedThemes: string[];
}

interface ExportConfig {
  format: 'json' | 'csv' | 'huggingface' | 'pdf' | 'docx';
  includeMessages: boolean;
  includeProfiles: boolean;
  includeOpinions: boolean;
  includeThemes: boolean;
  includeMetrics: boolean;
  includeMemory: boolean;
  includeRawLLM: boolean;
  anonymize: boolean;
  prettyPrint: boolean;
}
```

## Consequences

### Positive

- **Comprehensive analysis** covers all major insight types (sentiment, themes, opinions)
- **AI-powered findings** reduce manual analysis effort
- **Multiple export formats** serve different downstream needs
- **Cross-simulation analytics** provide organizational insights
- **Publication-ready reports** support research workflows

### Negative

- **AI analysis cost** adds LLM calls after simulation completes
- **Complex export options** may overwhelm casual users
- **Theme clustering** requires semantic embeddings (compute cost)

### Implementation Notes

1. **Key Findings**: Uses ResultsExtractor from ADR-010
2. **Sentiment Analysis**: Can use lightweight local model or LLM
3. **Theme Clustering**: Use embeddings + HDBSCAN or similar
4. **Export Formats**: Use pandas for CSV, jinja2 for PDF templates
5. **HuggingFace**: Use `datasets` library for parquet export
